# ExifTool Runtime Exception

Status: accepted for the first media-transcoding release.

ExifTool is included because it provides broad metadata inspection and bounded
metadata rewrite coverage that is not uniformly available through FFmpeg,
MediaInfo, MKVToolNix, GPAC, or Bento4 alone.

## Bounds

- Revaer may invoke ExifTool only through a typed adapter.
- The adapter may pass only allowlisted arguments generated by Revaer.
- ExifTool execution must happen inside a managed workspace.
- ExifTool must not mutate source files directly.
- ExifTool must not leave source-adjacent backup or sidecar clutter.
- Release evidence must record the exact package version, package source,
  checksum or package digest, license text, attribution, and security-review
  result for the image digest being published.

## Review Triggers

- Any additional source-form runtime utility requires a separate ADR.
- Any direct source mutation by ExifTool requires a separate threat model and
  ADR.
- Any packaging mode outside Alpine packages requires refreshed source,
  checksum, license, and notice evidence.
